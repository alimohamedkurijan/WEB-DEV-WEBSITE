üîÄ What are Conditions & Logic?

Conditions and logic allow your code to make decisions and respond differently based on various situations. Think of it as the brain of your program - just like you make decisions in life, your code can make decisions too!

üìö Key Points:

  • Decision Making: Code can choose different paths based on conditions
  • User Input: Respond differently to different user actions
  • Data Validation: Check if data meets certain criteria
  • Dynamic Behavior: Create interactive and responsive applications
Simple Conditional Example 
let age = 18;

if (age >= 18) {
    console.log("You can vote!");
} else {
    console.log("Too young to vote.");
}

// Multiple conditions
if (age < 13) {
    console.log("You're a child");
} else if (age < 20) {
    console.log("You're a teenager");
} else {
    console.log("You're an adult");
}

What happens:

1. Checks if age is 18 or older

2. Prints "You can vote!"

3. Then checks age ranges

4. Prints "You're an adult"

‚öñÔ∏è Comparison Operators

Comparison operators let you compare values and create conditions. Understanding the difference between loose and strict equality is crucial for writing reliable code.

üîç Comparison Types:

  • Loose Equality: == (compares values, converts types)
  • Strict Equality: === (compares values AND types)
  • Inequality: != and !=
  • Relational: >, <, >=, <=
Comparison Operators Examples 
// Loose vs Strict Equality
console.log(5 == "5");   // true (converts string to number)
console.log(5 === "5");  // false (different types)

console.log(0 == false); // true (both are falsy)
console.log(0 === false); // false (different types)

// Relational operators
let score = 85;
console.log(score > 80);   // true
console.log(score >= 85);  // true
console.log(score < 90);   // true
console.log(score <= 85);  // true

// Inequality
console.log(score != 80);  // true
console.log(score !== "85"); // true (different types)

üí° Pro Tip:

Always use === and != instead of == and !=. This prevents unexpected type conversions and makes your code more predictable and reliable.

‚öôÔ∏è Function Declaration & Calling

Functions are reusable blocks of code that can accept input (parameters) and return output. They help you organize code and avoid repetition.

üîß Function Components:

  • Declaration: Define what the function does
  • Parameters: Input values the function accepts
  • Arguments: Actual values passed when calling
  • Return: Output value the function sends back
Function Basics 
// Function declaration
function greet(name) {
    console.log("Hello, " + name + "!");
}

// Function call with argument
greet("Ali");        // Output: Hello, Ali!
greet("Sarah");      // Output: Hello, Sarah!

// Function with multiple parameters
function introduce(firstName, lastName, age) {
    console.log("Hi, I'm " + firstName + " " + lastName);
    console.log("I'm " + age + " years old");
}

introduce("Ali", "Ahmed", 25);
// Output: Hi, I'm Ali Ahmed
//         I'm 25 years old

üîÑ Return Statements

The return statement sends a value back from a function. This is different from console.log() - returned values can be stored in variables and used elsewhere in your code.

üì§ Return vs Console.log:

  • Return: Sends value back to calling code
  • Console.log: Displays value in console (for debugging)
  • Storing Results: Returned values can be assigned to variables
  • Reusability: Returned values can be used in calculations
Return Statements Examples 
// Function with return
function add(a, b) {
    return a + b;
}

// Store returned value
let result = add(5, 3);
console.log(result); // 8

// Use returned value in calculations
let sum1 = add(10, 20);
let sum2 = add(5, 15);
let total = sum1 + sum2;
console.log(total); // 50

// Function with conditional return
function checkAge(age) {
    if (age >= 18) {
        return "Adult";
    } else {
        return "Minor";
    }
}

let status = checkAge(20);
console.log(status); // "Adult"

// Multiple return points
function getGrade(score) {
    if (score >= 90) return "A";
    if (score >= 80) return "B";
    if (score >= 70) return "C";
    if (score >= 60) return "D";
    return "F";
}

console.log(getGrade(85)); // "B"

üéØ Building Function-Based Apps

Now let's combine everything we've learned to create a complete application that uses functions, conditions, and user input to provide personalized responses.

üõ†Ô∏è App Building Strategy:

  • Input Functions: Collect and validate user data
  • Processing Functions: Apply logic and make decisions
  • Output Functions: Display results to users
  • Main Function: Orchestrate the entire process
Complete Function-Based App 
// Function to get user input
function getUserInfo() {
    let name = prompt("What's your name?");
    let age = prompt("How old are you?");
    return { name: name, age: parseInt(age) };
}

// Function to check age-based permissions
function checkPermissions(age) {
    if (age >= 18) {
        return "You can vote, drive, and access all features!";
    } else if (age >= 16) {
        return "You can drive with supervision and access limited features.";
    } else if (age >= 13) {
        return "You can access basic features with parental consent.";
    } else {
        return "You can access child-friendly features only.";
    }
}

// Function to create personalized message
function createMessage(name, age, permissions) {
    return `Hello ${name}! You are ${age} years old.\n${permissions}`;
}

// Main function that runs the app
function runApp() {
    // Step 1: Get user input
    let userInfo = getUserInfo();
    
    // Step 2: Process the data
    let permissions = checkPermissions(userInfo.age);
    
    // Step 3: Create output
    let message = createMessage(userInfo.name, userInfo.age, permissions);
    
    // Step 4: Display results
    alert(message);
    console.log("App completed for:", userInfo.name);
}

// Run the application
runApp();

üí° Pro Tip:

Break down complex applications into smaller, focused functions. Each function should have a single responsibility. This makes your code easier to read, test, and maintain.
