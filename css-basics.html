üéØ What is CSS?

CSS = Cascading Style Sheets

CSS explains how HTML elements are styled and presented. While HTML provides the structure, CSS provides the visual design.

üí° Real-world Analogy

HTML is the building structure, CSS is the paint, decor, and layout that makes it beautiful and functional.

You'll see a quick demo comparing unstyled vs styled versions of a page to understand the power of CSS.

2

üß© Inline vs Internal vs External CSS

There are three main ways to apply CSS to your HTML:

Inline CSS

Style="" inside HTML tags

<p style="color: red;">Text</p>

Internal CSS

<style> inside <head>

<style>
  p { color: red; }
</style>

External CSS

.css file linked with <link>

<link rel="stylesheet" href="style.css">

We'll explore the pros and cons of each method and when to use which approach.

3

üéØ Selectors, Properties, Values

CSS follows a simple structure: selector { property: value; }

üîç Types of Selectors

  • Tag selectors: p { } - targets all <p> elements
  • Class selectors: .myClass { } - targets elements with class="myClass"
  • ID selectors: #myId { } - targets element with id="myId"

We'll explain cascading and the importance of specificity, then add examples to your "About Me" page.

üñåÔ∏è Styling Essentials

Learn the fundamental CSS properties that will transform your pages:

üé® Colors & Backgrounds

  • color - text color
  • background-color - background color
  • HEX values (#ff0000) and named colors (red)

üìù Typography

  • font-family - font type
  • font-size - text size
  • line-height - spacing between lines

üî≤ Borders & Shapes

  • border - border styling
  • border-radius - rounded corners

üìè Spacing

  • margin - outer spacing
  • padding - inner spacing

We'll show side-by-side differences using browser live editing, and challenge you to apply at least 4 of these properties to your HTML content.

5

üÜö Class vs ID

Understanding when to use classes vs IDs is crucial for effective CSS:

üì¶ Classes

Reusable styling

  • Multiple elements can have the same class
  • Use for common styling patterns
  • Example: .box, .title

üÜî IDs

Unique identifiers

  • Only one element can have a specific ID
  • Use for unique styling or JavaScript targeting
  • Example: #header, #main-content

We'll provide code samples and highlight sections using both approaches to demonstrate the differences.
